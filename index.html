课程目标Course Goals

你将在三次课程中学会什么？What Will You Learn?

这门课不从“我要做哪种游戏”开始，而是从“我想让玩家经历什么、学会什么”开始。技术服务于设计判断，而不是替代它。

This course does not begin with “what genre should I make?” It begins with “what should the player experience and learn?” Technology serves design judgment—it does not replace it.

01 · ANALYZE

读懂游戏如何运作Read How Games Work

从情感处境、玩家学习、系统反馈和挑战因素分析游戏，而不只停留在故事或美术。

Analyze emotional situations, player learning, system feedback, and challenge factors—not only story or visual style.

02 · TRANSFORM

把经验转化为系统Transform Experience into Systems

找到新手误解、高手判断和关键数据，把真实经验或领域知识转化为玩家行动。

Find novice misunderstandings, expert judgments, and key data, then turn real experience or knowledge into player actions.

03 · DESIGN

设计学习与挑战Design Learning and Challenge

用反馈帮助玩家学习，用少数变量组合出2–3个递进而有意义的挑战。

Help players learn through feedback, then combine a few variables into 2–3 meaningful progressive challenges.

04 · BUILD

与AI协作开发Build with an AI Agent

向AI准确表达系统结构，检查它的理解，并分阶段完成网站、游戏和开发记录。

Communicate the system clearly, check the Agent's understanding, and build the site, game, and log in stages.

05 · TEST

通过试玩做决定Make Decisions through Playtesting

观察玩家是否真正学会预期能力，并根据测试结果保留、修改或删除设计。

Observe whether players learn the intended ability, then keep, change, or remove design elements based on evidence.

06 · REFLECT

留下真实的设计过程Document the Real Process

记录失败、取舍和AI的影响，说明哪些关键判断仍然属于你。

Record failures, tradeoffs, and AI influence, showing which important judgments remain yours.

学习流程Learning Journey

三次课程，一个完整项目Three Sessions, One Complete Project

每次课程都会产生下一阶段需要的材料。请按顺序完成，并在进入开发前确认自己的设计判断。

Each session produces the materials needed for the next. Work in sequence and confirm your own design judgment before development begins.

Session 1

从 INSIDE 学习游戏分析Learn Game Analysis through INSIDE

拆解情感处境、玩家学习循环、反馈与挑战空间，并提取一个30秒最小原型。

Break down emotional situations, learning loops, feedback, and challenge space, then extract a 30-second minimum prototype.

→
Session 2

设计你自己的游戏系统Design Your Own Game System

完成六步互动工作簿，在Canva中绘制系统图，并下载供AI开发使用的Markdown摘要。

Complete the six-step workbook, draw a system graph in Canva, and download a Markdown brief for AI development.

→
Session 3

与AI Agent共同开发Develop with an AI Agent

建立网站和可玩核心循环，增加挑战、试玩迭代、记录反思，并打包离线展览版本。

Build the website and core loop, add challenges, playtest, reflect, and package an offline exhibition build.

→

最终成果Final Outcomes

你带走的不只是一个游戏文件More Than a Game File

最终项目是一套能够展示、继续开发，也能够回看设计过程的完整材料。

The final project is a complete body of work that can be exhibited, continued, and used to understand your design process.

项目网站Project Website

清楚介绍创意、领域知识、系统设计、开发过程和游戏入口。

A clear presentation of the idea, domain knowledge, system, process, and game.

可玩游戏Playable Game

一个完整的核心循环，以及2–3个由变量生成的递进挑战。

One complete core loop and 2–3 progressive variable-based challenges.

系统设计图System Graph

连接数据、玩家操作、系统计算、反馈和挑战的可视化模型。

A visual model connecting data, player actions, calculations, feedback, and challenges.

开发日志Development Log

按时间记录AI协作、实现失败、测试结果、设计决定和重要修改。

A timeline of AI collaboration, failures, tests, decisions, and important changes.

学生反思Student Reflection

说明AI如何影响方向、哪些决定属于自己，以及游戏是否仍表达原目标。

How AI affected the direction, which decisions remained yours, and whether the game kept its goal.

展览提交包Exhibition Package

包含离线版本、源文件、README、manifest和预览图的完整ZIP。

A complete ZIP with the offline build, source, README, manifest, and preview image.

AI可以帮助你实现游戏，但不能替你决定这个游戏为什么值得被设计。 AI can help you build the game, but it cannot decide why the game is worth designing.

你负责选择经验、判断系统是否准确、亲自试玩、决定取舍并完成反思。AI Agent负责协助整理、实现、测试和记录。

You choose the experience, judge whether the system is accurate, playtest, make tradeoffs, and reflect. The AI Agent assists with organization, implementation, testing, and documentation.
