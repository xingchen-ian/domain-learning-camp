Development process

This page is a public summary of the build log. The full, raw agent-development-log.md is in the project root and shows every change, every failed idea, and every decision in order.

Timeline

  • 2026-07-15 · Session 1

    Project setup & first playable (Level 1)

    Read the design brief and system graph. Locked in: 2D top-down view, 5 clubs, traps reset ball to last good position with +1 stroke penalty. Built the project structure, the website shell (Home, Domain & System, Development, Play), and Level 1 (Par 3, 100 yards, no traps) end-to-end. Player can aim, pick a club, set power, swing, and either hole out or be told why they missed.

  • Next · planned

    Add Levels 2 & 3 with traps

    Add water and sand traps plus trees. Make the canvas handle the longer holes (Level 2 = 200 yds, Level 3 = 330 yds). Verify win/lose flow on each level.

  • Later · planned

    Polish & submission

    Sounds for water splash and sand thud, score screen, submission manifest, preview screenshot, ZIP packaging, and final exhibition test on a static server.

My decisions (Zoe)

  • Picked 2D top-down instead of side view — the traps and trees are easier to read.
  • Picked 5 clubs — driver / 3-wood / 5-iron / 7-iron / wedge.
  • Picked "ball resets to last good position +1 stroke" for traps — feels fairer than restarting the hole.
