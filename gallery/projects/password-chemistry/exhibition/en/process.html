Development Process

An honest project build timeline — including mistakes, revisions, and the decisions behind AI suggestions that were accepted, modified, or rejected. This page is a web-summary of agent-development-log.md in the project root.

2026-07-15 · Step 1
Establish Project Identity
Student: ELSA ZHANGYIQING. Project title: PASSWORD. Domain: Chemistry & Escape Room. Tool: AI coding assistant (Track A — no build step, pure HTML/CSS/JS + Canvas 2D). Core learning shift recorded: "Novices think an escape room is about getting out; experts know it's about reading the environment, developing logic, and using tools comprehensively."
2026-07-15 · Step 2
Design Brief & System Diagram Locked
Wrote the development brief, covering identity, design summary, the observe→judge→act→read feedback→adjust loop, the environment/player/system data model, rules, three challenge presets, web-game technical dependencies, the integrated project website requirements, and the human–AI log protocol. The system diagram (made in Canva) was exported into the design summary, giving implementation a clear direction.
2026-07-15 · Step 3
Scaffold Project Structure
Created the expected file structure per the brief: exhibition/ for the final static site (index.html, game.html, domain.html, process.html, and assets/ holding CSS, JS, and images), with source/ as a mirror for editable source. Initialized agent-development-log.md with the required header and interaction record 01.
2026-07-15 · Step 4
Game Engine Implementation (Track A)
Built the playable game in pure HTML + CSS + Canvas 2D + vanilla JS — no build step, no CDN dependencies. The game implements the full observe→judge→act→feedback→adjust loop:
  • Four-direction movement + circular collision detection
  • 3 challenge presets with different time limits, sprite counts, and recipe complexity
  • Lab bench: explicit mix→filter→heat→evaporate steps
  • Sprite AI: patrol (Level 1) and chase (Levels 2 and 3)
  • Health bar, countdown timer, chemical counters, screen shake
  • Purple smoke and flash particle effects on reaction success/failure
  • Win/lose screens with health lost, time used, and chemical stats
2026-07-15 · Step 5
Build the Website Around the Game
Built the four required web pages in the same Track A static style:
  • Home — game concept, core learning shift, level overview
  • Domain & Systems — domain knowledge, novice misconceptions, system diagram, feedback transitions
  • Development Process — this timeline, a development log summary
  • Play — direct entry into the playable game, with level selection
All navigation and assets use relative paths so the build output can be dropped straight into an exhibition subfolder.
2026-07-15 · Step 6
Visual Restyle — Surreal Cute
Student feedback: the original dark sci-fi style felt off. The entire visual language was redesigned as surreal cute: soft lavender/mint/pink palette, rounded candy walls, checkerboard floors, a cat-eared protagonist with blush, big-eyed purple ghost sprites, potion bottles marked with stars, magic portal exits, heart/star/bubble particle effects, and floating background decorations. NPC speed was lowered at the same time, an objective hint bar was added, and the lab-bench popup was rebuilt (with numbered recipe steps and a progress bar). All four website pages were updated in sync.
2026-07-20 · Step 7
Audio, Visual Polish & Story Immersion
The student requested several enhancements to boost the game's appeal and readability:
  • BGM background music: Web Audio API synthesized 8-bit melody, inspired by Snake. A cheerful C-major pentatonic loop that plays during the game and lab bench, and stops on menus and end screens.
  • Item scale 2.5×: chemical bottles and notes were enlarged so players can read formula labels even at exhibition resolution (1920×1080).
  • Lab bench cutified: restyled as a warm wood-table lab bench with floating equipment decorations (🧪⚗️🔬🧫), rounded buttons, gradient background, and hover animations.
  • Celebration system: a successful experiment triggers a special celebration sound (triangle-wave rising chord) + a full-screen "SUCCEED!" animation (rainbow gradient shimmer), 30 star emojis, and 60 colorful confetti particles.
  • Story opening animation: before each level a story cutscene plays — featuring a CSS-drawn chemist protagonist (round head, goggles, lab coat, flask, with blink, arm-swing, and float animations). Text is revealed with a typewriter effect. The countdown only starts after the player clicks "Start Mission". Each level has its own themed scene particles (hospital 💊🩺, factory 💧🏭, darkroom 📷🎞️).
2026-07-21 · Step 8
Bug Fix & Final Submission Review
  • Critical bug fix: the addChemical() function had an operator-precedence error — !arr.indexOf(key) >= 0 is always true in JavaScript, so every chemical addition was treated as an error. Fixed to indexOf(key) === -1.
  • Data consistency sweep: unified all level names, times, and descriptions across 4 files (game.html, game.js, README.md, story-opening text). Fixed the Level 3 time inconsistency (some places showed 40s; the actual value is 60s).
  • Documentation updates: updated the README challenge-preset table, THIRD_PARTY.md (clarified Web Audio API synthesis), submission-manifest.json (removed the outdated "no sound effects" issue, added the bug-fix record and feature summary), and agent-development-log.md (added interaction record 04).
  • Level 3 time: extended from 40s to 60s to improve playability with 4 NPCs and the complex 4-step reaction.
