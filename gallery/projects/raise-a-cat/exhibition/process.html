Key Design Decisions

  1. Track A over Track B/C: The game is 2D and doesn't need 3D (Three.js) or complex module loading. Plain HTML/CSS/JS is the simplest and most reliable approach for exhibition on an iMac.
  2. Three distinct cat personalities: Rather than difficulty levels (easy/hard), each cat has different system variables — Shadow's sensitive stomach, Bounce's fast metabolism, Mimi's balanced stats. This directly supports the core learning shift about adapting to different cats.
  3. Web Audio API for sounds: Instead of external audio files, meow sounds are synthesized programmatically. This ensures the game works offline with zero external dependencies.
  4. Visual feedback over text-only: The cat's animations, tail flicking, dim lighting when sick, and playfulness glow make the feedback intuitive — players learn to read cat behavior, just like in real life.
  5. Stat interdependence: Health is not directly controlled — it's calculated from fullness, water, cleanliness, and sickness state. This teaches players that neglecting any aspect affects overall health.
