Visual & Mechanical Polish

2026-07-15 — Camellia Cai & WorkBuddy

Major aesthetic and gameplay overhaul based on playtesting feedback. Redesigned the visual theme to a pastel pink bakery style and refined all mechanics.

Visual Design

  • Pastel pink color scheme with decorative hearts, flowers, and baking tools
  • Illustrated ingredient icons (butter block, sugar cube, flour bag, baking powder on a spoon, vanilla flower, eggs, chocolate, milk carton)
  • Top-down mixing bowl that shows accumulated ingredients inside a circular bowl with a visible rim
  • End-result pastry drawing (cake/cookies/brownies) with color saturation tied to accuracy score
  • Baking-themed decorations: whisk, chef's hat, pot with steam, rolling pin, completed pastries
  • Pastel accent colors: purple, green, yellow, blue, mint, lavender dots
  • Website pages restyled to match the game's bakery theme
  • Stopwatch timer in top-left corner with counterclockwise hand and tick marks
  • Baking powder rendered as white powder on a metallic spoon

Gameplay Mechanics

  • Mixing changed to tap-space-N-times with countdown timer per difficulty (7/6/5s for 10-press, 5/4/3s for 5-press)
  • Oven changed to toggle on/off (press space once to start, once to stop)
  • All ingredients visible at once — player must reference recipe to pick correctly
  • Wrong ingredient selection triggers an error buzz with no penalty other than wasted time
  • Mixer and oven always visible as selectable appliances — player must click to activate
  • Level-specific ingredient sets (5 for Cake, 5 for Cookies, 7 for Brownies)

UI & Feedback

  • Compact two-column result screen with summary/measurements on the left, appliance results/issues on the right
  • Recipe panel shows completed (green), active (pink highlight), and upcoming steps
  • Real-time progress bar for mix press count with gradient fill
  • Circular countdown ring for mix time limit (green → orange → red)
  • Audio feedback: per-step jingles (perfect/okay/bad), win/loss fanfares, error buzz

Development Approach

This project follows Track A — No build step: the game is built with plain HTML, CSS, and JavaScript using the Canvas 2D API. No npm, Vite, or external libraries are required. The exhibition version is a fully self-contained static website.

Key Technical Decisions

  • Canvas 2D over Three.js: The game is fundamentally 2D (measuring cups, progress bars, buttons). Three.js would add unnecessary complexity.
  • Web Audio API for sounds: All jingles are generated procedurally with oscillators — no external audio files needed.
  • Single-page game: The playable game lives in game.html with all logic embedded for simplicity and portability.
  • Relative paths throughout: All links and assets use relative paths so the project works inside a gallery subfolder.
