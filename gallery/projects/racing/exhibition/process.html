Stage 1 — First Playable Version

July 15, 2026 · Project Initialization

Project Setup and Core Implementation

Goal: Build the smallest complete game loop that validates one observe → judge → act → feedback → adjust cycle.

What was built:

  • Top-down 2D racing game with scrolling road, lane markings, and grass shoulders
  • WASD driving controls: accelerate, brake, steer left/right
  • Space bar boost with limited energy that recharges
  • Police car that chases from behind — proximity drives siren volume and red/blue flash intensity
  • Traffic cars in three lanes that the player must dodge
  • Gas system: fuel depletes based on speed and boost usage, refilled by collecting gas tokens on the road
  • Car health: collisions cause damage, damage reduces top speed and increases gas drain
  • Smoke particle effects when the car is damaged
  • HUD showing health, gas, boost, speed, time remaining, distance progress, and police distance
  • Procedural audio via Web Audio API: siren (wah-wah effect), engine hum, crash noise, low gas alert, gas collection chime, win/lose fanfares
  • 3 challenge levels (Novice, Driver, Expert) that change traffic density, police speed, gas drain rate, and track length
  • Integrated website with Home, Play Game, and Process pages
  • Win condition: reach destination before time runs out
  • Lose conditions: time runs out, police catches you, run out of gas, car health reaches zero

Technical decisions:

  • Chose Track A (no build step): HTML + CSS + plain JS + Canvas 2D — no npm, no Vite, no external dependencies
  • All audio synthesized with Web Audio API — no audio files needed, works fully offline
  • All graphics hand-drawn with Canvas 2D primitives — no spritesheets or image assets
  • Game canvas is 960x640, CSS-scaled to fit the container for responsive display on iMac (1920x1080) and student laptops

Challenges solved:

  • Police chase balance: tied police distance directly to player speed — slow down = police gets closer, creating the core learning tension
  • Gas token visibility: tokens pulse and glow so they're visible against the road surface
  • Damage feedback chain: damage → slower speed → harder to escape police → need to collect gas → slow down → police gets closer — a multi-layered consequence system
  • Audio with no files: used OscillatorNode + GainNode with LFO modulation for the siren, buffer sources with noise for crashes

Student decisions: Awaiting playtest feedback. The student will test all three levels and provide reactions on difficulty, controls, and whether the learning shift is communicated.
