Development Process

A chronological record of how Wing Flight was built through human-AI collaboration.

Development Timeline

The timeline below is based on agent-development-log.md and records the actual development process — including failures and partial results.

Interaction 01 — Project Setup

Reading the Brief & Planning

Read the domain-learning brief. Identified the core learning shift (wind observation + energy evaluation), the observe → judge → act → feedback → adjust loop, and the three data categories (environment, player-controlled, system-calculated). Decided on Track A (no build step) since the game is 2D and Canvas 2D is sufficient.

Interaction 02 — Project Structure

File Structure & Website Skeleton

Created the required directory structure: exhibition/ with index.html, game.html, process.html, and assets/. Initialized agent-development-log.md with the project header. Created a shared style.css for consistent styling across all pages.

Interaction 03 — Game Engine Core

Canvas 2D Physics & Rendering

Implemented the core game engine in game.js: player physics (gravity, velocity, wind effects), energy system (drain on ascend, regen on dive, storm drain), collision detection (coins, energy pickups, land masses, storms), particle system for visual feedback, and a parallax cloud rendering system. Used Web Audio API for synthesized sound effects — no external audio files needed.

Interaction 04 — Level Design

Three Challenge Presets

Designed and implemented three levels that change system variables, not just visuals: Level 1 (First Flight) teaches basic controls with calm wind; Level 2 (Headwind Trial) forces wind observation and energy planning with strong headwinds and exploitable updrafts; Level 3 (Storm Alley) demands route planning with six storm obstacles and shifting wind. Each level has different time limits, starting energy, and minimum coin requirements.

Interaction 05 — HUD & Feedback

Wind Vane, Energy Bar, and Feedback Systems

Built the HUD: energy bar (color-coded green/yellow/red), wing strength indicator (3 segments), wind vane compass showing direction and strength, coin counter, timer, and distance progress bar. Implemented feedback overlays: screen shake for turbulence, red flash for route drift, red pulsing border for low energy, and a dark sky effect when inside storms. All feedback maps directly to the brief's feedback translation table.

Interaction 06 — Website Content

Home Page with Domain Knowledge

Built the home page with four sections: Game Idea (concept, player goal, learning shift), Domain Knowledge (novice vs. expert comparison, misconceptions), System Design (environment data, player data, feedback table, challenge presets), and navigation to the playable game. Styled cards and tables to make the domain knowledge readable for classmates who haven't seen the project.

Interaction 07 — Testing & Polish

Playtesting & Bug Fixes

Tested all three levels: verified win/lose conditions, coin collection, energy pickup, storm collision, land collision, timer expiry, and next-level progression. Fixed the wing strength toggle (changed from A/D keys to 1/2/3 number keys for clarity). Added the level-select menu with level descriptions and controls help. Confirmed the game runs from a static file server without Node.js or npm.

Interaction 08 — Energy Supply Enhancement

More Energy Pickups + New Large Energy Orb

Student requested more energy supplies. Added more regular energy orbs across all three levels (Level 1: 3→7, Level 2: 4→8, Level 3: 5→10). Created a new "large energy pickup" type: a bigger blue-cyan orb with a spinning sparkle ring and star icon that restores 50% energy (vs. 30% for regular). Placed large orbs at strategic chokepoints — before strong headwinds and between storm clusters — to give players a fair replenishment opportunity before difficult sections. Added a distinct ascending arpeggio sound effect for large pickups.

How AI Helped

  • Implementation: The AI agent wrote the entire game engine (game.js), CSS styles, and HTML pages based on the design brief.
  • Level Design: The AI agent designed the three challenge presets by varying wind zones, obstacles, energy, and time — following the brief's requirement that presets change system variables, not just visuals.
  • Feedback Systems: The AI agent mapped the brief's feedback translation table (flapping, speed drops, route drift) to concrete visual and audio effects in the game.
  • Technical Setup: The AI agent chose Track A (no build step) and used Web Audio API for synthesized sounds, ensuring the game runs offline without external dependencies.
  • Documentation: The AI agent maintained the development log and project documentation as required by the brief.

Key Design Decisions

  • Side-view over top-down: A side-view makes altitude, gravity, and energy management tangible — you can see yourself falling when energy runs out.
  • Wind zones instead of global wind: Different parts of the level have different wind, forcing the player to observe and adapt their route as they fly.
  • Dive = energy regen: This creates the natural "zigzag" pattern experts use in real upwind sailing — dive to build speed and energy, then ascend in bursts.
  • Three wing strengths: Light wings are efficient but can't fight strong wind; strong wings are powerful but drain energy fast. Players must match wing to conditions.
  • Synthesized audio: No external sound files — all effects are generated with Web Audio API, keeping the project self-contained.
