Squint

A small game about the hardest thing in painting: seeing light, not objects.

Domain · Painting  |  Student · Charlie. S  |  Track A (no build)

Game Idea

Concept. You are shown a lit scene. Instead of painting details, you block in the big shapes and assign each one a value from a limited 5-step grayscale (later, mixed colour). A "squint" view blurs your canvas against the reference so you can judge the masses the way a real painter does.

Player goal. Match the reference's value structure — not its outlines — using as few strokes as possible, before the stroke budget runs out.

Core learning shift. key idea "Painting isn't just copying objects. It's all about translating the light you see into objective values and shapes."

Domain Knowledge

Real-world domain

Painting from observation. The same object looks completely different under different light direction and temperature, so the painter must identify and understand the light before anything else. The student has painted since childhood and studies art in school.

Novice misconception

Beginners paint the picture in their head instead of what they actually see. They start with details (eyelashes, texture) instead of blocking in the big masses, and they assign colours from memory instead of observation.

Expert judgment

Experts squint constantly. They read the scene as a few large value shapes, judge each value against its neighbours, and only then refine. They know colour is decided by context, not by the object's "real" colour.

Why it becomes playable

The squint is the perfect game mechanic: blur removes detail and leaves only value relationships. The game makes the invisible (light) visible through pops, step-back, and an end-of-level critique.

System Design

System graph: environment data, player input, calculated results, exposed feedback, and the three level presets.

Environment data (read-only)

  • Reference scene (objects + light)
  • Light direction & temperature
  • Hidden true-value map
  • Stroke budget (int)
  • Level definition

Player-controlled data

  • Place / resize shape blocks (drag)
  • Assign value 1–5 (keys / palette)
  • Mix colour (Level 3 slider)
  • Squint (Q), Step-back (E), Flip (F)

System-calculated results

  • Per-region value match
  • Strokes used vs. budget
  • Squint-match similarity
  • AI critique score (int)
  • Win / lose state

Feedback translation

  • Region pop — a mismatched area glows red in squint mode.
  • Squint match — blurred canvases compared overlay/flip.
  • Step-back — canvas shrinks to test the whole from distance.

Challenge presets

The first reference (a lit sphere + box) is reused for the first three levels so the core skill is drilled before new subjects appear. After that, each level is a different, more interesting reference.
