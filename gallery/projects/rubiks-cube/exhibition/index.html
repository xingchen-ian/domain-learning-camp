Game Idea

3x3 Rubik's Cube is a learning game built around the real experience of solving a Rubik's Cube. Players start by experimenting with random face turns, then progress through three difficulty levels that gradually remove hints, encouraging them to recognize patterns and remember algorithms on their own.

Player goal: restore the cube to a fully solved state with as few moves and as little time as possible.

Core learning shift: moving from random turns to recognizing patterns and choosing the correct algorithm.

Domain

Solving the Rubik's Cube

The real challenge is not turning one color onto a face — that destroys parts of the cube that were already solved. The actual method (CFOP) builds the cube in stages: cross, F2L, OLL, PLL.

Misconception

Novice assumption

Beginners believe the keys should directly be turned onto the face each piece belongs to. In practice this random-turn approach almost always breaks the work that came before it.

Expert judgment

Pattern recognition

Experts read the cube as a small set of patterns (headlights, solved corners, edge pairs) and pick the algorithm that fits the current pattern, not the desired end state.

Why playable

Information & memory

The challenge of incomplete information, time, memory, accuracy and efficiency translates naturally into a game where the cube is the puzzle, the clock is the pressure, and the move log is the score.

System Design

The game exposes the three data roles described in the development brief — environment data, player-controlled data, and system-calculated results — on screen at all times so players can see how their actions change the system.

Environment data

Cube state

Colors and piece positions, the current pattern, and which face is selected / highlighted. This is the puzzle the player has to read.

Player input

Six faces + actions

Turn the 6 faces (R, L, U, D, F, B) by clicking or pressing the matching key. Shuffle, undo, redo, restart, and give-up are exposed as buttons.

Calculated

Time · moves · % solved

The system calculates elapsed time, total moves made, and percent of stickers that are on their correct face. These three numbers translate invisible progress into readable feedback.

Three levels

Progressive hint removal

Level 1 shows the full algorithm for the current stage. Level 2 shows a brief hint with no algorithm. Level 3 shows no hint and adds a 5-minute countdown.

Win & lose conditions

Win: all six faces show a uniform color (each of the 54 stickers matches the center of its face). The timer stops and the final time, move count, and level are shown.

Lose: the player chooses Give Up, or on Level 3 the 5-minute countdown reaches zero. In both cases the final attempt summary is shown and the player can restart.

Tech notes

The game is a static web app using a locally vendored copy of three.js 0.160.0. No build step, no CDNs, no internet connection needed at runtime. Open exhibition/index.html in a browser served by any static HTTP server and you're done.
