Development process

A truthful chronological timeline of how this project was built with an AI agent. Mirrors agent-development-log.md.

How the project was built

Human–AI collaboration. The student owns the design decisions; the AI agent implements, explains, and suggests.

  • AI agent used: WorkBuddy (the active development agent). The brief originally listed ChatGPT; the actual build was done with WorkBuddy.
  • Dependency track: Track A — no build step. Plain HTML, CSS, and Canvas 2D JavaScript.
  • Source of truth for this page: agent-development-log.md. Nothing here is invented or smoothed over.

Timeline

Interaction 01 · Session 1

Project initialized & first playable loop built

Read the brief and the system graph, restated the core learning shift and core loop, then scaffolded the project structure (exhibition/, source/). Chose Track A (no build) so the final build runs from a static folder. Implemented the full observe → judge → act → feedback → adjust loop: diver movement, oxygen, depth/pressure, equipment stability, species scanning with a real fact catalog, radar beeps, pressure mesh, and attack feedback.

Interaction 01 · Session 1

Three challenge presets added

Defined presets that differ through variables, not decoration: Reef Welcome (tutorial, shallow), Pressure Point (deep, equipment strain), and Big Fish, Calm Diver (the whale-shark lesson that delivers the core learning shift). Each changes equipment depth limit, oxygen rate, species pool, target count, and danger.

Interaction 01 · Session 1

Integrated website created

Built Home, Domain & System Design, Development Process, and Play Game pages with shared styling and Gallery-compatible relative paths. Wired the system graph into the design page.

Interaction 01 · Session 1

Local test pass

Served exhibition/ from a local static server and verified navigation, all three challenges, win/lose, restart, and the radar/pressure/attack feedback. Confirmed offline-capable relative paths.

Interaction 02 · Session 1

Redesigned for coral reef exploration & eco-health

Student requested a more intuitive experience: looking through coral reefs for animals and maintaining environmental health. Rebuilt the game world around a living coral reef — coral formations (brain, fan, tube, anemone, plate) with individual health, creatures that hide inside coral and emerge when the diver approaches, a pollution system where debris drifts and damages coral, and an eco-health meter that is a win condition. Added cleaning by swimming through pollution, coral glow for hidden creatures, coral bleaching when sick, and water murk when the ecosystem declines. Replaced the three challenge presets with Reef Welcome, Stressed Reef, and Predator Balance. Updated all website pages, README, and docs to match.

What the AI influenced

Design directions AI introduced

The radar-beeps-for-size (not danger) mechanic, and the penalty for startling harmless creatures, were AI suggestions that directly serve the learning shift. They are recorded as AI influence, not student authorship, until the student confirms.

Open questions for the student

Public-display consent for the manifest; development start/end dates; and whether the species facts read as accurate enough to keep. These are flagged in the log, not decided silently.

Reflection checkpoint

The first playable loop is complete. A stage reflection has been opened in the log and the student is asked to answer it — the AI does not fill it in.

Required student reflection (pending)
