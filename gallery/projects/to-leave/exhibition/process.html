Development Process

2026-07-21

Project Initialization & First Playable Build

Goal: Set up project structure and implement the smallest complete game loop (Observe → Judge → Act → Feedback → Adjust).

What happened:

  • Student provided the complete development brief.
  • Agent analyzed the brief and identified Track A (HTML/CSS/Canvas 2D, no build step) as the appropriate dependency track for a 2D packing puzzle.
  • Created project directory structure: exhibition/, source/.
  • Initialized agent-development-log.md.
  • Implemented core game engine with 3 challenge presets:
    • Case 1: The Wealthy Lady (talkative, 8-slot suitcase, 10 objects)
    • Case 2: The College Student (quiet, 6-slot backpack, 9 objects)
    • Case 3: The Elderly Man (silent, 7-slot travel bag, 10 objects)
  • Each challenge differs through variables: container size, character voice style, clue formats, object sets, and Need/Want distributions.
  • Implemented full interaction: examine objects, enter memories (Canvas-rendered scenes), pack/unpack, container capacity tracking, voice feedback, scoring.
  • Created project website with Home, Domain & Design, and Development Process pages.

AI influence: Agent designed the 3 challenge presets based on the brief's description. The memory visualization system (Canvas-drawn scenes) was an agent suggestion to make emotional weight experiential rather than just numerical.

Student decision: Student has not yet reviewed the first playable version. Feedback on character design, object selection, and gameplay feel is pending.

Open questions:

  • Does the scoring formula correctly reward the intended learning shift?
  • Are the 3 cases sufficiently different in challenge difficulty?
  • Is the memory viewing experience engaging enough to make players care about Want scores?

Note: This timeline will be updated after each meaningful development milestone. It will record failures and abandoned directions as honestly as successes. No events have been fabricated or smoothed.
