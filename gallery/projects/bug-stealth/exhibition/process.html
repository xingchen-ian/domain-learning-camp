Development Process

This timeline is a faithful summary of agent-development-log.md. It records what was built, what failed, and how the human–AI collaboration shaped the project.

Interaction 01 — Initialization

2026-07-15

Stage: Project setup.

Goal: Read the development brief + system graph and build the first playable version of Bug Stealth.

What happened: The student dropped the brief (plant-care.md), the spec (Bug Stealth.docx) and the system graph screenshot into the workspace. WorkBuddy read all three, summarized the core learning shift, the three difficulty tiers, the variable list and the HUD requirements.

AI influence: WorkBuddy chose Track A (no build step — plain HTML + CSS + Canvas 2D + JavaScript) because the brief mandates a build that runs from a teacher-hosted static server with no npm install and no internet. Three.js was not necessary for the gameplay the system graph actually describes.

Result: Project skeleton created, system graph and brief copied into the project root, agent-development-log.md initialized.

Interaction 02 — Game engine & levels

2026-07-15

Stage: First playable loop.

Goal: Implement the smallest complete observe → judge → act → feedback → adjust loop.

What happened: Wrote levels.js (the 3 challenge presets as pure data) and main.js (the engine: input, player movement, insect AI with patrol/alert/dive states, mirror rotation, ray–mirror reflection, plant growth, win/lose detection, HUD, overlays). game.html hosts the canvas and the control legend.

Key mechanic — light reflection: the sun emits a ray; for each mirror in the path the engine tests ray–segment intersection, computes the reflected direction r = d − 2(d·n)n, and only accepts the bounce when the angle of incidence is inside mirrorReflectionRange. The plant receives sunlight only when the final ray hits it.

Test & immediate result: Loaded locally — canvas renders, player walks and hides, insects patrol and dive, mirrors rotate, light path recomputes. Partially working: climb-to-win animation needs polish (see open questions).

Student decision / follow-up: Pending student confirmation of project title "Bug Stealth" and difficulty wording.

Interaction 03 — Project website

2026-07-15

Stage: Website + docs.

Goal: Build the integrated project website required by brief §9 and the root deliverable docs (README, THIRD_PARTY, manifest).

What happened: Created index.html (home), domain.html (domain knowledge), system.html (system graph + data model + win/lose + presets), process.html (this page), README.md, THIRD_PARTY.md and submission-manifest.json. All internal links use relative paths so the project works inside a Gallery subfolder.

Open questions / known issues

  • Climb-to-win transition is functional but visually simple — a dedicated climbing animation could be added later.
  • Audio (BGM, footsteps, wing sounds) is intentionally omitted in the first playable version to keep the file count low; it can be layered in via the Web Audio API without changing mechanics.
  • The student has not yet confirmed the public-display consent field for the submission manifest.
