DOMAIN & SYSTEM

How the game is built and how its parts relate

Core loop

The player moves through tile-based lab rooms, avoids or eliminates enemies, and reaches the green EXIT. The game is a single HTML/Canvas file (assets/game.js) with no build step and no external dependencies.

Systems

  • Movement & collision — tile grid, solid walls, doors, and cover.
  • Enemy AI — wander patrol, vision cone, line-of-sight blocked by walls/doors/cover, chase on sight.
  • Combat — melee (fists/knife/bat) and ranged weapons; contact-only death for the player.
  • Feedback — detection bar, health/danger HUD, blood on death.
  • Levels — three hand-authored grid layouts with a safe spawn room.

Design intent

The challenge for a new player is learning to read enemy perception and use the environment (doors, cover, sight-blocking furniture) rather than only clicking. Different weapons trade noise for speed, teaching weapon-choice strategy.

Built for the First Playable Game exhibition · plain HTML/CSS/Canvas (Track A)
